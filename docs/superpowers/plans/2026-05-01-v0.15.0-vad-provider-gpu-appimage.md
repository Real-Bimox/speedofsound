# v0.15.0 — Dedicated VAD Provider + GPU Sherpa + AppImage Distribution

> **Status (2026-05-01):** Plan implemented. VAD-15 closed (`VadProvider` enum, `VoiceModel.provider` widened to `SelectableProvider`). GPU JAR build via `scripts/build-gpu-jar.sh` (~400 MB, gitignored, reproducible). Both `make appimage` (CPU, ~201 MB) and `make appimage-gpu` (GPU, ~423 MB) produce working self-contained AppImages. CUDA runtime libs are NOT bundled — Sherpa expects them on the host; documented in script + README. End-to-end GPU smoke (CUDA actually engaged at runtime) pending host-side verification.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship VoiceStream v0.15.0 as a self-contained Linux AppImage with optional GPU acceleration via the prebuilt CUDA 12 / cuDNN 9 Sherpa ONNX bundle, and clean up VAD-15 by giving Silero its own provider type instead of borrowing `AsrProvider.SHERPA_WHISPER`.

**Architecture:**
- **VAD-15:** Add a `VadProvider` enum (initial value `SILERO`) implementing the existing `SelectableProvider` interface. Widen `VoiceModel.provider` to accept `SelectableProvider` so VAD models can declare their own provider without touching `AsrProvider`. ASR catalog routing (`getModelsForProvider(AsrProvider)`) keeps its enum-typed parameter and stays exhaustive — VAD models become invisible to ASR-flow callers, which is the desired isolation.
- **GPU bundle:** Sherpa publishes a prebuilt `sherpa-onnx-v1.12.33-cuda-12.x-cudnn-9.x-linux-x64-gpu.tar.bz2` containing GPU-aware native shared libraries. Extract its `libsherpa-onnx-jni.so` + `libonnxruntime.so` and repackage them into a JAR matching the existing `sherpa-onnx/native/linux-x64/` resource layout, then publish it as `sherpa-onnx-native-lib-linux-x64-gpu-v1.12.33.jar` in `core/libs/`. The Gradle build picks the GPU JAR when the `voicestream.gpu=true` Gradle property is set; otherwise it picks the existing CPU JAR. Runtime CUDA fallback in `SherpaOfflineAsr.kt` already gracefully demotes to CPU when CUDA init fails, so a GPU-built bundle on a non-GPU host still works.
- **AppImage:** `make appimage` already produces a near-AppImage from `jpackage` + `appimagetool`. Extend `scripts/build-appimage.sh` so that, when a GPU JAR is present in `core/libs/`, the resulting jpackage app-image includes the GPU runtime libs. The same script invoked without the GPU JAR continues to produce a CPU-only AppImage.

**Tech Stack:** Kotlin 2.3.10, JVM 25, Gradle 9.4.1, Sherpa ONNX 1.12.33 (CPU JAR + new GPU JAR), kotlin.test/JUnit Platform, detekt, jpackage (JDK 25), appimagetool, Apache Commons Compress (already on classpath via `ArchiveExtractor`).

## Stage 2: GPU Sherpa native lib bundle

> The Sherpa ONNX project publishes prebuilt CUDA artifacts as `.tar.bz2` bundles
> rather than as JARs. We download the official tarball, extract the two `.so`
> files, and repackage them into a JAR matching the layout the existing
> `sherpa-onnx-native-lib-linux-x64-v1.12.33.jar` uses, so the JNI loader picks
> them up unchanged.
>
> Runtime CUDA + cuDNN libraries are NOT bundled in the JAR — they ship in the
> AppImage in Stage 3 instead, so the host doesn't need a system-wide CUDA
> install.

### Task 6: Write `scripts/build-gpu-jar.sh`

**Files:**
- Create: `scripts/build-gpu-jar.sh`

- [ ] **Step 1: Write the script**

```bash
#!/bin/bash
# Downloads the Sherpa ONNX CUDA tarball, extracts the GPU-aware shared libraries,
# and repackages them into a JAR matching the layout the JVM JNI loader expects.
# Outputs core/libs/sherpa-onnx-native-lib-linux-x64-gpu-v${VERSION}.jar plus
# the raw CUDA runtime libs in core/libs/cuda-runtime-linux-x64/ for the
# AppImage to bundle later.
#
# Usage: scripts/build-gpu-jar.sh
# Requires: curl, tar, bzip2, zip, sha256sum

set -euo pipefail

VERSION="v1.12.33"
SHERPA_BASE="https://github.com/k2-fsa/sherpa-onnx/releases/download/${VERSION}"
TARBALL_NAME="sherpa-onnx-${VERSION}-cuda-12.x-cudnn-9.x-linux-x64-gpu.tar.bz2"
EXPECTED_SHA256=""  # populated on first run from checksum.txt — see below

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
ROOT_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
LIBS_DIR="$ROOT_DIR/core/libs"
WORK_DIR="$(mktemp -d)"
trap 'rm -rf "$WORK_DIR"' EXIT

OUTPUT_JAR="$LIBS_DIR/sherpa-onnx-native-lib-linux-x64-gpu-${VERSION}.jar"
OUTPUT_RUNTIME_DIR="$LIBS_DIR/cuda-runtime-linux-x64"

# Refuse to overwrite if both already exist (idempotent re-runs)
if [ -f "$OUTPUT_JAR" ] && [ -d "$OUTPUT_RUNTIME_DIR" ]; then
    echo "GPU JAR + runtime already present:"
    echo "  $OUTPUT_JAR"
    echo "  $OUTPUT_RUNTIME_DIR"
    echo "Delete to rebuild."
    exit 0
fi

# Resolve checksum from the official checksum.txt if not pinned above
if [ -z "$EXPECTED_SHA256" ]; then
    echo "Fetching checksum.txt to resolve $TARBALL_NAME hash..."
    EXPECTED_SHA256="$(curl -sL "$SHERPA_BASE/checksum.txt" | awk -v t="$TARBALL_NAME" '$2==t{print $1}')"
    if [ -z "$EXPECTED_SHA256" ]; then
        echo "Could not find $TARBALL_NAME in $SHERPA_BASE/checksum.txt" >&2
        exit 1
    fi
    echo "Resolved checksum: $EXPECTED_SHA256"
fi

# Download
TARBALL_PATH="$WORK_DIR/$TARBALL_NAME"
echo "Downloading $TARBALL_NAME ..."
curl -fL "$SHERPA_BASE/$TARBALL_NAME" -o "$TARBALL_PATH"

# Verify
echo "Verifying SHA256..."
echo "$EXPECTED_SHA256  $TARBALL_PATH" | sha256sum -c -

# Extract
echo "Extracting..."
tar -xjf "$TARBALL_PATH" -C "$WORK_DIR"
EXTRACTED_DIR="$WORK_DIR/sherpa-onnx-${VERSION}-cuda-12.x-cudnn-9.x-linux-x64-gpu"
if [ ! -d "$EXTRACTED_DIR/lib" ]; then
    echo "Unexpected tarball layout — no lib/ directory" >&2
    ls "$EXTRACTED_DIR"
    exit 1
fi

# Build JAR with the two GPU-aware libs at the path the JNI loader expects
JAR_STAGE="$WORK_DIR/jar"
mkdir -p "$JAR_STAGE/sherpa-onnx/native/linux-x64"
cp "$EXTRACTED_DIR/lib/libsherpa-onnx-jni.so" "$JAR_STAGE/sherpa-onnx/native/linux-x64/"
cp "$EXTRACTED_DIR/lib/libonnxruntime.so" "$JAR_STAGE/sherpa-onnx/native/linux-x64/"
mkdir -p "$JAR_STAGE/META-INF"
cat > "$JAR_STAGE/META-INF/MANIFEST.MF" << 'EOM'
Manifest-Version: 1.0
Created-By: VoiceStream build-gpu-jar.sh
EOM
( cd "$JAR_STAGE" && zip -r "$OUTPUT_JAR" META-INF sherpa-onnx )

# Stage CUDA runtime libs separately for the AppImage step. These are NOT in the JAR
# because they are large (>500 MB) and only need to be on LD_LIBRARY_PATH at runtime.
mkdir -p "$OUTPUT_RUNTIME_DIR"
shopt -s nullglob
for lib in "$EXTRACTED_DIR"/lib/libcudnn*.so* "$EXTRACTED_DIR"/lib/libcudart*.so* \
           "$EXTRACTED_DIR"/lib/libcublas*.so* "$EXTRACTED_DIR"/lib/libcufft*.so* \
           "$EXTRACTED_DIR"/lib/libcurand*.so* "$EXTRACTED_DIR"/lib/libnvJitLink*.so*; do
    cp -P "$lib" "$OUTPUT_RUNTIME_DIR/"
done

echo "Created: $OUTPUT_JAR"
echo "Staged runtime libs: $OUTPUT_RUNTIME_DIR"
ls -lh "$OUTPUT_JAR" "$OUTPUT_RUNTIME_DIR"
```

```bash
# Bundle CUDA runtime libs when a GPU build was performed (core/libs/cuda-runtime-linux-x64/ exists)
CUDA_RUNTIME_DIR="$ROOT_DIR/core/libs/cuda-runtime-linux-x64"
if [ -d "$CUDA_RUNTIME_DIR" ]; then
    echo "Bundling CUDA runtime from $CUDA_RUNTIME_DIR..."
    mkdir -p "$APP_DIR/lib/runtime/lib"
    cp -P "$CUDA_RUNTIME_DIR"/*.so* "$APP_DIR/lib/runtime/lib/"
    # Wrap the launcher so LD_LIBRARY_PATH includes our bundled CUDA libs
    LAUNCHER="$APP_DIR/bin/voicestream"
    LAUNCHER_REAL="$APP_DIR/bin/voicestream.real"
    if [ -f "$LAUNCHER" ] && [ ! -f "$LAUNCHER_REAL" ]; then
        mv "$LAUNCHER" "$LAUNCHER_REAL"
        cat > "$LAUNCHER" << 'WRAP'
#!/bin/bash
DIR="$(dirname "$(readlink -f "$0")")"
export LD_LIBRARY_PATH="$DIR/../lib/runtime/lib:${LD_LIBRARY_PATH:-}"
exec "$DIR/voicestream.real" "$@"
WRAP
        chmod +x "$LAUNCHER"
    fi
    OUTPUT_SUFFIX="-gpu"
else
    OUTPUT_SUFFIX=""
fi

# Build the AppImage, APPIMAGE_EXTRACT_AND_RUN=1 avoids a FUSE dependency
OUTPUT="$OUTPUT_DIR/voicestream-${VERSION}${OUTPUT_SUFFIX}-${ARCH}.AppImage"
```
